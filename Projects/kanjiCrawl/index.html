Kanji Crawl

This project uses a remote API to download necessary data to play a small memorization game. Kanji is one of the three written scripts of the Japanese Language, it contains several thousand logographs that each represent words or concepts. This project downloads kanji as JSON data, creates C# objects, and serializes those objects to XML documents for local storage. We do this to avoid unneeded stress on the remote server we obtain our data from, as well as to track the user's progress locally.

Some Examples.

First, we look at the data we will need to download from the API. For our game we simply need the kanji's character and meanings; however, we will also take the readings as those may be useful in late versions of this program. Currently, players are given a random kanji definition from the current set. They must navigate around several incorrect kanji and find the one that corresponds to the given definition.

The data available for each kanji.

The kanji are categorized by grade, and the player will select the grade they wish to practice before playing. Once the user selects a grade, a corresponding XML file is deserialized into a KanjiList Object. If the KanjiList is empty, we must download the list from the API.

Once the list is downloaded and placed into a KanjiList object, it is written to the corresponding XML file through an XML serializer. Next, a new object is instantiated using the recently downloaded data (or data loaded from an existing file if a set exists for a category). This object handles the basic game loop, which is pretty straightforward. Several random kanji are scattered across a grid, and one is randomly selected as the target. When the user hits the save and quit button, their accuracy for memorizing a given kanji is updated. When the user next loads the game, they will be able to see that accuracy in the upper-left corner.
