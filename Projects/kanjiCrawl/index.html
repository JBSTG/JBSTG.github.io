Kanji Crawl

This project uses data from here to create a tool for learning the Japanese Kanji writing system. The writing system uses thousands of logographic characters and forms the majority of Japanese scripts. It is very difficult to learn, so I wanted to create something to help.

Some Examples.

The data acquired from the API is stored as XML files prior to being loaded into a SQLite database. I build the SQLite database on my end to spare the user first-time setup. I prefer this approach as I can make changes to the database or the XML files very easily as well.

A rough diagram of the database.

The project uses a game format to help users learn, they use the arrows keys to navigate to the correct kanji. I feel like this provides a low-stress experience for users, although I may add an optional time limit in the future.

Current Gameplay

I also used this project as an opportunity to experiment with WPF's templating tools. Users can switch between styles stored in a resource dictionary. This is saved upon restarting using .NET settings configration utilities and is also updated immediately as the top-level view model is notified immediately. Currently, users can toggle styles and gridline visibility.

Demonstrating configuration changes

However, my primary purpose for this project is to further familiarize myself with the Model-View-ViewModel design pattern often used with WPF. The purpose of this design pattern is to improve UI content updates via data-binding, assist in debugging by centralizing logic, and improve loose-coupling by removing the Xaml file's dependency on its code-behind (Which also assists in debugging). The following image is my code behind for the game view shown above, an empty code behind does not necessarily demonstrate good design, and is sometimes not even possible. However, for this project I have been able to closely adhere to MVVM and observe some of its strengths and weaknesses first-hand.
